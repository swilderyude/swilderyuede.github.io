# 八股文：计算机研究生面试常见问题

## 1. 计算机网络

### 1.1 OSI 七层模型

**问题**：请解释 OSI 七层模型及其各层的功能。

**答案**：
- **物理层**：负责数据的物理传输，如比特流的传输
- **数据链路层**：负责数据帧的传输，如 MAC 地址、差错检测
- **网络层**：负责数据包的路由和转发，如 IP 地址、路由选择
- **传输层**：负责端到端的通信，如 TCP、UDP 协议
- **会话层**：负责建立、维护和终止会话
- **表示层**：负责数据的格式转换，如加密、压缩
- **应用层**：为应用程序提供服务，如 HTTP、FTP、SMTP 等

### 1.2 TCP 和 UDP 的区别

**问题**：请比较 TCP 和 UDP 的区别。

**答案**：
| 特性 | TCP | UDP |
|------|-----|-----|
| 连接性 | 面向连接 | 无连接 |
| 可靠性 | 可靠，通过确认、重传机制 | 不可靠，无确认机制 |
| 传输速度 | 较慢 | 较快 |
| 拥塞控制 | 有 | 无 |
| 流量控制 | 有 | 无 |
| 适用场景 | 网页浏览、文件传输 | 视频通话、在线游戏 |

### 1.3 HTTP 和 HTTPS 的区别

**问题**：请解释 HTTP 和 HTTPS 的区别。

**答案**：
- **安全性**：HTTPS 是 HTTP 的安全版本，使用 SSL/TLS 加密
- **端口**：HTTP 使用 80 端口，HTTPS 使用 443 端口
- **URL 格式**：HTTP 以 http:// 开头，HTTPS 以 https:// 开头
- **性能**：HTTPS 由于加密和解密过程，性能略低于 HTTP
- **证书**：HTTPS 需要 CA 证书，HTTP 不需要

## 2. 操作系统

### 2.1 进程和线程的区别

**问题**：请解释进程和线程的区别。

**答案**：
- **资源分配**：进程是资源分配的基本单位，线程是 CPU 调度的基本单位
- **内存空间**：进程有独立的内存空间，线程共享进程的内存空间
- **切换开销**：进程切换开销大，线程切换开销小
- **通信方式**：进程间通信需要特殊机制（如管道、消息队列），线程间通信通过共享内存
- **稳定性**：进程崩溃不影响其他进程，线程崩溃可能导致整个进程崩溃

### 2.2 死锁的条件和预防

**问题**：请解释死锁的四个必要条件和预防方法。

**答案**：
**四个必要条件**：
- 互斥条件：资源不能被多个进程同时占用
- 请求与保持条件：进程持有资源的同时又请求新资源
- 不剥夺条件：资源只能由持有它的进程主动释放
- 循环等待条件：多个进程形成环状的资源等待链

**预防方法**：
- 破坏互斥条件：使用共享资源
- 破坏请求与保持条件：一次性申请所有资源
- 破坏不剥夺条件：允许抢占资源
- 破坏循环等待条件：按顺序申请资源

### 2.3 页面置换算法

**问题**：请介绍常见的页面置换算法。

**答案**：
- **FIFO**（先进先出）：淘汰最早进入内存的页面
- **LRU**（最近最少使用）：淘汰最近最少使用的页面
- **OPT**（最佳）：淘汰未来最长时间不使用的页面（理论上的最优算法）
- **LFU**（最不经常使用）：淘汰访问次数最少的页面

## 3. 数据结构与算法

### 3.1 时间复杂度和空间复杂度

**问题**：请解释时间复杂度和空间复杂度的概念。

**答案**：
- **时间复杂度**：衡量算法执行时间随输入规模增长的变化趋势，常用大 O 表示法，如 O(1)、O(log n)、O(n)、O(n log n)、O(n²) 等
- **空间复杂度**：衡量算法所需存储空间随输入规模增长的变化趋势，同样使用大 O 表示法

### 3.2 排序算法

**问题**：请比较常见排序算法的时间复杂度和空间复杂度。

**答案**：
| 排序算法 | 平均时间复杂度 | 最坏时间复杂度 | 最好时间复杂度 | 空间复杂度 | 稳定性 |
|---------|---------------|---------------|---------------|-----------|--------|
| 冒泡排序 | O(n²) | O(n²) | O(n) | O(1) | 稳定 |
| 选择排序 | O(n²) | O(n²) | O(n²) | O(1) | 不稳定 |
| 插入排序 | O(n²) | O(n²) | O(n) | O(1) | 稳定 |
| 归并排序 | O(n log n) | O(n log n) | O(n log n) | O(n) | 稳定 |
| 快速排序 | O(n log n) | O(n²) | O(n log n) | O(log n) | 不稳定 |
| 堆排序 | O(n log n) | O(n log n) | O(n log n) | O(1) | 不稳定 |

### 3.3 二叉树的遍历

**问题**：请解释二叉树的前序、中序和后序遍历。

**答案**：
- **前序遍历**：根节点 → 左子树 → 右子树
- **中序遍历**：左子树 → 根节点 → 右子树
- **后序遍历**：左子树 → 右子树 → 根节点

## 4. 数据库

### 4.1 关系型数据库和 NoSQL 数据库的区别

**问题**：请比较关系型数据库和 NoSQL 数据库的区别。

**答案**：
| 特性 | 关系型数据库 | NoSQL 数据库 |
|------|-------------|-------------|
| 数据模型 | 表格结构，有固定 schema | 灵活的数据模型，如文档、键值、列族等 |
| 事务支持 | 支持 ACID 事务 | 部分支持，通常为最终一致性 |
| 扩展性 | 垂直扩展 | 水平扩展 |
| 查询语言 | SQL | 各自的查询语言或 API |
| 适用场景 | 结构化数据，需要事务支持 | 非结构化数据，高并发场景 |
| 代表产品 | MySQL, PostgreSQL, Oracle | MongoDB, Redis, Cassandra |

### 4.2 数据库索引

**问题**：请解释数据库索引的作用和类型。

**答案**：
**作用**：
- 加快数据查询速度
- 提高数据检索效率
- 减少数据库的 I/O 操作

**类型**：
- **主键索引**：唯一标识表中的每一行
- **唯一索引**：确保索引列的值唯一
- **普通索引**：加快数据查询速度
- **组合索引**：基于多个列的索引
- **全文索引**：用于全文搜索

### 4.3 SQL 优化

**问题**：请介绍 SQL 优化的方法。

**答案**：
- 使用索引：为经常查询的列创建索引
- 避免全表扫描：使用 WHERE 子句过滤数据
- 优化 JOIN 操作：使用合适的 JOIN 类型，避免笛卡尔积
- 避免使用 SELECT *：只查询需要的列
- 使用 LIMIT 限制返回行数
- 避免在 WHERE 子句中使用函数：会导致索引失效
- 合理使用 GROUP BY 和 ORDER BY

## 6. 系统设计

### 6.1 高并发系统设计

**问题**：请介绍高并发系统的设计原则。

**答案**：
- 负载均衡：将请求分发到多个服务器
- 缓存：使用 Redis、Memcached 等缓存热点数据
- 数据库优化：分库分表、读写分离
- 异步处理：使用消息队列处理非实时任务
- 服务拆分：微服务架构
- 限流：防止系统过载
- 降级：在系统压力大时，关闭非核心功能
- 监控：实时监控系统状态

### 6.2 分布式系统

**问题**：请解释分布式系统的 CAP 理论。

**答案**：
- **一致性**（Consistency）：所有节点在同一时间看到相同的数据
- **可用性**（Availability）：系统在任何时候都能响应请求
- **分区容错性**（Partition Tolerance）：系统在网络分区的情况下仍然能正常工作

CAP 理论指出，在分布式系统中，这三个特性最多只能同时满足两个。

### 6.3 缓存策略

**问题**：请介绍常见的缓存策略。

**答案**：
- **Cache-Aside**：应用程序直接管理缓存，先查缓存，缓存未命中则查数据库并更新缓存
- **Read-Through**：缓存服务负责从数据库加载数据
- **Write-Through**：缓存服务负责将数据同步写入数据库
- **Write-Back**：缓存服务先更新缓存，然后异步写入数据库
